工作流项目介绍

行业背景

在众多软件开发的需求中，有一类比较特殊且常见的业务流程，比如在学习中奖学金申请、在公司中向领导请假审批，它们需要有人发起申请，然后有人审批，因此常被称为审批业务流程。例如：下图中养老人员申请入住养老院的业务流程：养老人员发起入住申请后，需要主管、副院长、院长等多人审批人，每个审批人还可以进行同意、拒绝、驳回等操作，不同的操作对应不同的程序流向，因此整个业务流程显得比较复杂，而这样的业务流程如果使用传统的硬编码方式来实现，代码实现逻辑比较复杂，且很难维护。因此就需要更好的方式来开发类似的审批业务流程，而工作流技术就是这样一种技术。

工作流技术作为信息技术领域一项重要技术。在企业数字化转型中可以帮助企业提高效率、降低成本，同时改善客户体验；在企业产品中可以预定规则，从而提供企业产品的正确性和效率，进而提升产品优势；在企业内部协作和沟通中，还可以帮助团队成员传递任务、通知和信息，确保沟通畅通，任务不会被遗漏。总体来说，工作流技术在各个企业或者行业中都是一项重要的技术，能够有效的帮助企业自动化业务、提高效率、降低成本，以及快速的适应市场的变化。

工作流技术应用场景

工作流技术从198X年被提出发展至今，以及渗透发展到各行各业，比如企业服务、电商、制造业等，但是目前应用最广的还是企业服务行业，其OA、ERP、CRM等产品基本都包含工作流技术的应用，而OA（企业办公室自动化软件）相关的产品几乎是必用工作流技术，比如下图就是一个OA系统：

别看OA系统界面上的功能菜单比较少，但是其可以新建的处理流程确是非常多，甚至还可以无限增加，而且每个流程的逻辑都不一样，这都需要工作流技术来处理，因此OA系统是看起来简约，做起来很复杂的一类系统。

工作流技术前景

  • 企业服务行业是工作流技术的发源地
  • 工作流在各行各业中都有应用

市场规模及预测

  • 2023年工作流技术的市场规模达到70亿元左右，同比增长25%。

  • 预计2023-2032年工作流将迎来快速增长，到2032年市场规模达千亿。

课程背景

正是基于工作流技术的广泛应用和未来市场发展趋势的需要，研发了本《工作流组件》课程及项目，填补市场空白的技术课程，来帮助学员提升就业优势。

工作流组件既是课程也是传智自主研发的一个成熟且轻量级的开源项目（开源名：Hiss）。无论是单体架构还是微服务架构都适用。该组件旨在简化业务流程的设计、实施和管理，进而降低企业在工作流领域的使用和开发成本。该组件基于 Actviti 7 引擎进行二次开发，对 Actviti 7 在流程设计器、表单设计器等方面的缺陷进行了改进，并在底层通信、事件机制等方面进行了优化。这使得 Activiti 7 更加适用于微服务集成，符合微服务标准。

本着“让学员带着真本事到企业，能帮助企业解决问题”的目的，工作流组件课提供全面【流程引擎+低代码】的整合方案、及相关的工具。旨在让学员学完本课程不光有知识储备（主流的解决方案），还有高效的工具（Hiss开源项目），真正做到走进企业时是有本事的、有竞争力的！

业务架构

工作流组件可最终交付的内容分为一个组件库，两个系统端：

  • 组件库：提供了流程设计、低代码表单设计等相关的组件，可嵌入hiss-server、hiss-client两个系统中提高UI组件复用性
    • design-ui：提供了开发者、业务员可使用的流程设计器、状态图、办理页等功能UI
    • form-ui：提供业务员可使用的表单设计器、数据管理页等功能UI
  • 流程中心（hiss-center）：主要集中管理多租户的流程、实例、表单等数据
  • 业务系统（hiss-client）：泛指与流程中心进入的系统，在这些系统中可以直接嵌入流程办理、数据管理等相关的功能

功能大纲

  • 流程中心的功能包括应用管理、用户管理、流程管理、实例管理、表单管理、快速集成等功能，这些功能中除了应用管理、用户管理之外的功能都可以集成到业务系统中
  • 在流程管理中办理页面还提供了前加签、后加签、跳转等22个办理功能

基本概念

Activiti是一款主流的工作流 流程引擎，再此我们就先从工作流、流程引擎概念说起。

什么是工作流

工作流（ Workflow ）：是一项将系统任务或操作抽取，再按照预定业务流程进行组织、协调、并自动化执行的技术。

我们也可以简单理解==工作流是一种简化审批业务流程开发的技术==，其主要是解决一些业务场景硬编码方式的成本高、效率低、变更慢等问题的技术。

复杂的业务流程，实现复杂度比较高，因此开发周期长，开发成本高，效率也低，如果发生需求变更，对之前复杂的代码进行修改，同样需要更长的时间，因此响应变更的速度呢也表现得慢。

工作流技术来实现类似复杂的业务，只需要集成流程引擎、画出流程中使用的表单、最后画出整个业务流程图，交给流程引擎3步，即可完成需求的开发。在整个实现过程中，流程引擎是一个三方框架，不需要我们过多编码，而画表单、画流程就是在页面中进行一些拖拉拽的操作。因此整个过程下来我们编码量极少，需要的开发时间也极少，通过此方式呢就轻松的解决了开发周期长、成本高、效率低的问题，如果需求变更了，工作流技术只需要修复之前的表单或图即可，因此变更响应速度自然也必硬编码开发的速度要快。

什么是流程引擎

流程引擎：是用于驱动或控制业务流程每一步自动执行的代码框架。

目前市面上主流的流程引擎框架有jBPMN、Activiti、Flowable、Camunda4款，其中Activiti开源度最大，因此是当下主流的首选流程引擎。

本课程则选择Activiti7作为我们的学习流程引擎，主要因素有：

  • Activiti开源度大，企业使用多
  • Activiti成熟度好，文档资料完善
  • 学习完成Activiti之后，可以快速上手Flowable、Camunda等基于Activitii二次开源的框架

Activiti介绍

Activiti：是一个开源的轻量级工作流和业务流程管理与执行的引擎。2010年基于jBPM4实现首次开源，旨在帮助企业自动化和优化各种业务流程。

官网地址：https://www.activiti.org/

Activiti 的主要特点和优势：

  • 开源和跨平台：Activiti 是一个开源项目，可以在各种操作系统和平台上运行，提供灵活的部署选项。

  • 高度可定制：Activiti 提供了可视化的流程设计器，使用户能够轻松设计和定制各种业务流程。流程定义可以基于 BPMN 2.0 标准进行创建。

  • 强大的任务管理：Activiti 允许用户定义任务、分配任务给不同的用户或组，并支持任务的优先级和截止日期。

  • 工作流引擎：Activiti 提供了强大的工作流引擎，能够自动执行流程、处理任务、执行规则等。它支持顺序流、条件流、并行网关等流程控制元素。

  • 事件驱动：Activiti 支持事件驱动的流程，可以在特定的事件发生时触发流程的不同分支。

  • 可扩展性：Activiti 允许开发人员通过编写自定义 Java 类来扩展其功能，满足特定的业务需求。

  • 数据持久化：Activiti 使用关系型数据库来存储流程定义、流程实例、任务等信息，保证了数据的持久性和可靠性。

  • 集成支持：Activiti 可以与其他技术和系统集成，如 Spring、RESTful Web Services、JavaEE 等。

  • 监控和报告：Activiti 提供了流程实例的监控、报告和分析功能，使用户能够了解流程的状态和性能。
